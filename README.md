Features

Binary Protocol Parsing вҖ” Custom TCP/TLS server that decodes Teltonika Codec 8 & Codec 8 Extended binary protocol in real-time
Live Vehicle Tracking вҖ” Real-time position updates via Socket.IO with sub-second latency
Trip Detection вҖ” Automatic trip start/stop detection with route history and mileage calculation
Geofencing вҖ” Configurable geofence zones with push notification alerts on entry/exit
OBD Diagnostics вҖ” DTC (Diagnostic Trouble Code) event parsing and display from CAN bus data
Driver Management вҖ” Driver profiles linked to vehicles with activity tracking
Maintenance Tracking вҖ” Service intervals, reminders, and maintenance history per vehicle
Audit Logging вҖ” Full audit trail for compliance and fleet accountability
Multi-Tenant Architecture вҖ” Tenant isolation with JWT authentication and role-based access control (admin, manager, driver)

Teltonika Protocol

The IoT server implements a full binary parser for:


Codec 8 вҖ” Standard AVL data packets with GPS, I/O elements, and timestamps
Codec 8 Extended вҖ” Extended format supporting 2-byte I/O element IDs for OBD and CAN bus data


Each TCP connection handles device authentication via IMEI handshake, followed by continuous AVL data streaming with server-side acknowledgment.

Deployment

The platform runs on Fly.io in the Amsterdam region with:


Auto-scaling for the TCP ingestion server
Persistent PostgreSQL via Fly Postgres
Redis for session management and real-time pub/sub


Context

This project started as a practical solution for a real fleet of ~10 waste collection vehicles at Retim/Nimu Dragos in TimiИҷoara, Romania. The hardware integration (GPS modules, RFID antennas, cameras, Raspberry Pi controllers) was done hands-on вҖ” from wiring and mounting in the trucks to configuring Teltonika devices and connecting to the company's server infrastructure.
